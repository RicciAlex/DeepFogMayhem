タイトル: Pixel Legend
使用言語 : C++
開発期間 : 2022年6月～2022年9月(3か月)
ジャンル : 2Dシューティングゲーム


・説明
初めてC++で制作したゲームでした。
「Cuphead」と「Realm of the Mad God」というゲームを参考にしました。
8つあるステージから選択し、ボスと戦うゲームです。
クリア時間、選択した難易度と残った体力によってスコアが変わります。
ステージごとランキングがありますので、目的はハイスコアを目指すことです。
最初のボスScilla(スキュラ)はギリシャ神話の怪物をもとにして作りました。
ジークムント・フロイトの「生と死ドライブ」いわゆる「Eros」と「Thanatos」
を基にして、2つ目のボスを作りました。
最後に3つ目のボスから最後のボスまで七つの大罪を基にして作りました。
特にダンテ・アリギエーリの「神曲」の「煉獄」の同じ順番にあります。


・工夫した点・苦労した点
チュートリアル:
操作が分かりにくいと思いましたので、チュートリアルを作った時、
操作を説明するアニメーションを作りました。
元々普通の画像を使おうと思っていましたが、クラスメイトにプレイしてもらいまして、
操作分かりにくいと言われましたら、アニメーションを作ることにしたほうがいいと思いました。

当たり判定の管理:
元々当たり判定の処理は弾のクラスの中にありましたが、様々な問題がありましたので、
ヒットボックスのクラスを作ったほうが使いやすいと思いました。
新しいクラスを作りましたら、同じオブジェクトにヒットボックス複数を追加しましたり、
戦闘中ヒットボックスを消しましたり、追加しましたりすることができますので、とても便利になったと思います。

スケジュール:
初めてのC++のゲームでしたので、様々なミスをしてしまいましたので、バグが発生したことがあります。
それで思ったより時間がかかりまして、決まったスケジュールに比べて、かなり遅れてしまいました。
絵を描くのがかなり苦手ですので、特にテクスチャを作るには時間がかかりました。


・反省点
テクスチャアニメーションの管理:
テクスチャアニメーションの管理を改善することが出来ると思います。
特にアニメーションがループする場合、このゲームで最初のパターンに戻ります。
間違いなくそれを変えたほうがいいと思います。
それにアニメーションを外部ファイルで管理したほうがいいかもしれないと思います。

メニュークラス:
このゲームに色々なメニューがありますが、メニュークラスがありませんので、
ボタンの管理は少し複雑かもしれないと思います。
メニュークラスを作ったほうがよかったと思いますので、次のゲームに必ず実装してみようと思います。
